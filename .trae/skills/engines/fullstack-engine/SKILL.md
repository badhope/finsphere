## 📚 Embedded Knowledge Base

### Architecture Best Practices

**Three-Tier Standard Architecture**:
```
┌─────────────────────────────────────────┐
│  Frontend (React/Next.js/Vue)           │
│  ├── Components                         │
│  ├── Hooks/Composables                  │
│  ├── State Management                   │
│  └── API Clients                        │
├─────────────────────────────────────────┤
│  Backend API                            │
│  ├── Controllers/Routes                 │
│  ├── Services/Business Logic            │
│  ├── Validation Middleware              │
│  └── Auth & Permissions                 │
├─────────────────────────────────────────┤
│  Data Layer                             │
│  ├── Database Access (ORM)              │
│  ├── Migrations                         │
│  └── Caching Layer                      │
└─────────────────────────────────────────┘
```

### Docker Best Practices
- Use official base images with specific versions
- Multi-stage builds for minimal image size
- Non-root user for security
- Proper layer caching optimization
- Health checks for container orchestration
- `.dockerignore` to exclude build artifacts

### Production Security Checklist
- ✅ No `.env` files committed
- ✅ All secrets in environment variables
- ✅ Helmet.js or equivalent security headers
- ✅ Rate limiting on public endpoints
- ✅ Input validation on ALL endpoints
- ✅ Parameterized queries (no SQL injection)
- ✅ CORS properly configured
- ✅ HTTPS only in production
- ✅ Password hashing with salt (bcrypt/argon2)
- ✅ No debug stack traces in production

### Performance Optimization
- Database query indexing for common patterns
- Frontend bundle analysis and code splitting
- Proper HTTP caching headers
- Redis caching for frequent queries
- Connection pooling for databases
- Lazy loading for routes and components
- Image optimization and CDN delivery
