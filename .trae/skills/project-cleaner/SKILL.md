# Project Cleaner

## Description
智能清理项目中的多余文件、重复内容和未使用的资源。自动识别并安全删除冗余文件，优化项目结构。

## Details

### 功能特性
- 检测并删除重复文件
- 识别未使用的依赖和资源
- 清理临时文件和缓存
- 移除空目录
- 检测冗余的文档文件
- 安全删除（需确认）

### 检测规则

#### 1. 重复文件检测
- 内容完全相同的文件
- 文件名相似但内容相同的文件
- 备份文件（*.bak, *.backup, *.old）
- 临时文件（*.tmp, *.temp, *.swp）

#### 2. 未使用资源检测
- 未被引用的图片、字体文件
- 未被导入的模块
- 未使用的CSS类
- 未使用的配置文件

#### 3. 冗余文档检测
- 内容重复的README文件
- 过时的文档
- 空的文档文件
- 与主文档内容重叠的文件

#### 4. 项目特定清理
- node_modules（如果存在package-lock.json）
- .cache 目录
- dist/build 目录（如果不需要版本控制）
- .DS_Store, Thumbs.db
- IDE配置文件（可选）

### 安全机制
1. **预览模式**：先列出将要删除的文件，不实际删除
2. **确认机制**：重要文件需要用户确认
3. **备份选项**：删除前可选备份
4. **白名单**：保护重要文件不被误删
5. **撤销支持**：记录删除操作，支持恢复

### 使用场景
- 项目重构后清理
- 发布前优化
- 定期维护清理
- 合并分支后清理冲突文件

### 执行流程
1. 扫描项目目录结构
2. 应用检测规则识别候选文件
3. 分类展示清理建议
4. 用户确认后执行删除
5. 生成清理报告

### 白名单文件（不删除）
- LICENSE, LICENSE.*
- README.md, README.*
- CHANGELOG.md
- CONTRIBUTING.md
- SECURITY.md
- .gitignore
- package.json, package-lock.json
- 重要配置文件
