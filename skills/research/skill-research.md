# Research Skill

Systematic research methodology for investigating topics and making informed decisions.

## When to Use

- User needs to research a technology or tool
- Need to evaluate options before making decisions
- Want to understand a concept or domain
- Need to prepare a research brief or report
- Investigating problems or evaluating solutions

## Research Workflow

### 1. Prepare Research Brief

Use `prepare-research-brief` to structure the research:

**Components:**
- Research background and purpose
- Specific objectives and key questions
- Scope (inclusions/exclusions)
- Research methodology
- Information sources
- Deliverables and timeline
- Uncertainties and risks

### 2. Conduct Research

**Information Gathering:**
1. Official documentation and primary sources
2. GitHub repositories and code examples
3. Technical blogs and tutorials
4. Academic papers and research
5. Expert opinions and interviews

**Analysis Methods:**
1. Literature review
2. Comparative analysis
3. POC/Proof of concept
4. Expert consultation

### 3. Evaluate and Conclude

**Evaluation Framework:**

| Criterion | Weight | Option A | Option B | Option C |
|-----------|--------|----------|----------|----------|
| Functionality | 30% | - | - | - |
| Maturity | 25% | - | - | - |
| Performance | 20% | - | - | - |
| Cost | 15% | - | - | - |
| Risk | 10% | - | - | - |

**Decision Criteria:**
- Must-have requirements (go/no-go)
- Weighted scoring comparison
- Risk-adjusted recommendation

## Common Research Scenarios

### Technology Evaluation
- Compare frameworks/libraries
- Assess maturity and community
- Evaluate performance characteristics
- Consider learning curve

### Problem Investigation
- Understand root causes
- Research solutions
- Evaluate approaches
- Recommend action plan

### Market/Landscape Analysis
- Identify options
- Compare features
- Assess trade-offs
- Make recommendations

## Best Practices

1. **Define scope clearly**: Avoid research creep
2. **Use multiple sources**: Triangulate information
3. **Document findings**: Record what you learn
4. **Identify uncertainties**: Be clear about gaps
5. **Focus on actionable insights**: Research should inform decisions
