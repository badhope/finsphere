## 1. 项目概述

### 1.1 项目定位
为 AI Skill & Prompt Repository 项目打造一个高度视觉化、交互丰富的展示网站，兼具美观性与实用性，提供项目介绍、功能展示、内容导航三大核心功能。

### 1.2 核心设计理念
- **网页与内容分离**: 网站作为内容导航和展示渠道，不承载实际内容
- **渐进式披露**: 通过动画和交互引导用户逐步深入了解
- **所见即所得**: 所见元素皆可交互跳转

## 6. 响应式设计

### 6.1 断点设计

| 设备 | 断点 | 布局 |
|------|------|------|
| Mobile | < 640px | 单列，简化导航 |
| Tablet | 640-1024px | 双列，折叠菜单 |
| Desktop | 1024-1440px | 正常布局 |
| Large | > 1440px | 扩展间距 |

### 6.2 适配策略

**Mobile优先**:
- 3D效果降级为2D动画
- 粒子数量减少
- 简化导航为汉堡菜单

**性能优化**:
- 图片懒加载
- 动画暂停于后台标签
- 根据设备性能动态调整

## 7. 性能优化方案

### 7.1 前端性能

| 优化项 | 实现方式 | 目标指标 |
|--------|----------|----------|
| 首屏加载 | SSR + 预加载 | FCP < 1.5s |
| 动画性能 | GPU加速，will-change | 60fps |
| 图片 | WebP + 懒加载 | LCP < 2.5s |
| 代码分割 | Dynamic Import | 减少初始包 |
| 缓存 | 浏览器缓存 + CDN | 二次访问 < 0.5s |

### 7.2 后端性能

| 优化项 | 实现方式 | 目标指标 |
|--------|----------|----------|
| API响应 | 缓存 + 异步 | P99 < 200ms |
| 静态资源 | CDN分发 | 全球 < 100ms |
| 数据库 | 索引 + 查询优化 | 查询 < 50ms |
