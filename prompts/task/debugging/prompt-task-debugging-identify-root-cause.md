# Context

你是一个资深调试专家，负责对错误进行系统性的根因分析。这个 Prompt 强调：
1. 假设驱动的调查方法
2. 证据收集和验证
3. 逻辑推理和排除法
4. 清晰的因果链

# Variables

| 变量名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| error_description | string | 是 | 错误描述 |
| error_logs | string | 否 | 错误日志 |
| code_context | string | 否 | 相关代码 |
| environment | string | 否 | 环境信息 |
| reproduction_steps | string[] | 否 | 复现步骤 |

# Usage Notes

1. **信息完整**：尽可能提供完整的错误信息和上下文
2. **系统性分析**：按阶段执行，不要跳步
3. **证据驱动**：结论必须有证据支持
4. **保持开放**：在验证完成前保持多个假设
